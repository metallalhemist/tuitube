# Telegram menu disabled navigation and path redaction fixes

**Date:** 2026-05-01 17:03
**Files:** src/adapters/telegram/menus/download-menu.ts, src/adapters/telegram/menus/download-menu.test.ts, src/adapters/telegram/result-sender.ts, src/adapters/telegram/result-sender.test.ts
**Severity:** medium

## Problem

Two review findings needed fixes:

- The Telegram `Другие форматы` root button used `Menu.submenu`, so `@grammyjs/menu` scheduled navigation before custom middleware could reject an empty alternate-container menu.
- Telegram result-sender log redaction could leak unquoted local file paths with spaces, for example preserving a private title after redacting only the first path segment.

## Root Cause

`@grammyjs/menu` implements `submenu` by calling `ctx.menu.nav()` before running supplied middleware. That made the availability check too late to prevent navigation.

The redactor had regexes for quoted paths and whitespace-delimited unquoted paths, but unquoted paths with spaces are not whitespace-delimited. Media/transcript file paths often end in known extensions, so extension-aware matching is needed before the generic whitespace fallback.

## Solution

Changed the root `Другие форматы` button from `submenu` to `text`, then manually calls `ctx.menu.nav(DOWNLOAD_CONTAINER_MENU_ID)` only after `getOtherVideoFormatContainers` returns at least one target. Added `[FIX]` debug logging for the disabled no-container path.

Extended Telegram error sanitization with extension-aware `file://`, POSIX, and Windows path patterns before the existing generic path redactors. Added regression tests for the disabled menu callback and unquoted spaced path redaction.

## Prevention

- Avoid `Menu.submenu` when navigation depends on runtime availability checks; use a plain menu button and call `ctx.menu.nav()` after validation.
- Redaction tests should include quoted paths, unquoted paths, paths with spaces, URLs, and token-bearing strings.
- Review log-safety fixes with realistic Node/Telegram error message shapes, not only idealized quoted paths.
