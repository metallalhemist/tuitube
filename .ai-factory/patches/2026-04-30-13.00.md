# No-recode download plans for MP4-first format selection

**Date:** 2026-04-30 13:00
**Files:** src/core/format-selection.ts, src/core/types.ts, src/core/policy/download-policy.ts, src/core/services/video-download-service.ts, src/integrations/yt-dlp.ts, src/index.tsx, src/utils.ts, src/adapters/telegram/*
**Severity:** high

## Problem

Selected video formats could generate `--recode-video <ext>` from a `format_id#ext` value. On a one-core server this can trigger CPU-heavy video re-encoding and make downloads too slow. The UI also exposed raw-ish format choices instead of normalized no-recode download plans.

## Root Cause

The format value suffix after `#` was treated as a recode target instead of a final container marker. The old selection logic also appended `+bestaudio` when `acodec` was missing, which is unsafe because missing `acodec` in yt-dlp metadata does not reliably mean video-only. Format grouping happened too close to UI presentation, so raw DASH/adaptive streams could leak into choices.

## Solution

Added central `DownloadPlan[]` construction with generic metadata classification, safe merge container selection, size aggregation, and deduplication by container/height. MP4 options now use ready MP4 files or known-safe H264/AAC merges only. WEBM merges are limited to known-safe VP8/VP9/AV1 plus Opus/Vorbis pairs. `downloadFormatArgs()` now emits `--merge-output-format` only for explicit `video+audio` plans and never emits `--recode-video`.

Telegram menus now show MP4 plans on the first screen and use nested dynamic container and quality menus for other formats. The Termcast dropdown is driven by the same plans. Logging with `[FIX]` records chosen plans and yt-dlp argument mode.

## Prevention

- Keep format compatibility decisions in `src/core/format-selection.ts`.
- Do not infer video-only from missing `acodec`; require explicit metadata or safe adaptive merge handling.
- Treat `#container` as a plan marker, not a recode instruction.
- Add tests for classifier behavior, merge container allowlists, dynamic containers, and argument generation.
- Keep a regression check that runtime source does not contain active `--recode-video`.
